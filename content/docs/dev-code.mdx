## 代码操作

```bash
git clone git@gitlab.com:kong0/next-maker.git
```

建议是从 gitlab fork 到新项目仓库，然后克隆新的仓库代码。

新克隆的代码仓库，建议 3 个远程仓库：

- `lab` 是 gitlab 远程仓库，从 gitlab 克隆的仓库
- `hub` 是 github 远程仓库，方便 vercel 部署
- `ups` 是 gitlab 的上游仓库，方便同步 Next Maker 的基础功能

```bash
git remote add lab https://gitlab.com/kong0/[repo-name].git
git remote add hub https://github.com/kong0xyz/[repo-name].git
git remote add ups git@gitlab.com:kong0/next-maker.git
```

### 同步上游仓库

```bash
git fetch ups
git merge ups/main
git push hub
```

```bash

```

```bash

```

